# Good First Issues in Team Repos

After completing your core onboarding tasks, look for "good-first-issue" labels in these repos:

## How to Find Them

1. Go to the organization's GitHub: https://github.com/Cooperation-org
2. Browse repositories
3. Check Issues tab for "good-first-issue" label
4. Or use GitHub search: `org:Cooperation-org label:"good-first-issue" is:open`

## Before You Start

- Read the repo's README
- Check for CONTRIBUTING.md
- Look at recent merged PRs to see the style
- Set up local dev environment
- Ask in #dev if anything is unclear

## Types of Good First Issues

### Documentation
- Fix typos
- Add missing setup steps
- Improve error messages
- Add examples to README

### Code
- Add tests
- Fix linting issues
- Refactor for clarity
- Small bug fixes

### Tooling
- Improve dev setup
- Add helper scripts
- Better error handling

## Creating Good First Issues

Once you're comfortable, help create good-first-issues for future interns:
- Label them "good-first-issue"
- Provide context and links
- Describe success criteria
- Estimate: "should take 1-2 hours"
