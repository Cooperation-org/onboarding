# Developer Track

For interns and contributors focusing on engineering and technical work.

## Suggested Path

### Week 1: Setup & Understanding

**Required Reading** (read these first, no need to create issues):
- [Playing on an Organic Team](https://www.whatscookin.us/playing-on-an-organic-team/)
- [How to Jump on a Moving Project](https://www.whatscookin.us/how-to-jump-on-a-moving-project/)
- [ROLES.md](https://github.com/Cooperation-org/coop-projects/blob/main/ROLES.md) in coop-projects repo

**Tasks** (create issues using these templates):
- [001] Create Team Calendar Invite
- [008] Report in Standup
- [101] Setup Local Dev Environment
- [007] Ask Questions Usefully (when you hit issues)

### Week 2: Contributing Code

**Tasks:**
PICK a project 1st from [projects](https://github.com/Cooperation-org/projects)
- [102] Make Your First PR
- [005] Leave Breadcrumbs (improve README/docs based on setup experience)
- [105] Find and Fix an Issue
- [006] Seek Critique in Public (share your PR before submitting)

### Week 3: LinkedTrust Integration

**Tasks:**
- [002] Make Claim on dev.linkedtrust.us
- [003] Attest to Teammate's Claim
- [103] Use LinkedTrust SDK
- [104] Use create-trust-app
- [009] Review Teammate's Work

### Week 4: Real Work & Attestations

**Tasks:**
- [004] Make Real Claim on linkedtrust.us
- Pick up "good-first-issue" tasks from real projects
- Get peer reviews on your work
- Request attestations for skills demonstrated

## Completion

You're done when:
- All core tasks completed (001-009, 101-105)
- At least 3 merged PRs in team repos
- At least 2 peer reviews received
- Demonstrated proficiency with LinkedTrust SDK
- Published certificate on linkedtrust.us
