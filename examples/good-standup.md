# Standup Examples

## Good

```
**Yesterday**: finished tasks 001-003, link to issues in thread
**Today**: starting the breadcrumbs PR for linkedtrust docs
**Blockers**: none
```

```
**Yesterday**: got local dev running but CORS error on login
**Today**: debugging, posted in #dev https://slack.com/archives/C123/p456
**Blockers**: waiting on answer about CORS config
```

```
**Yesterday**: task 007 done, reviewed @sam's claim work
**Today**: not sure - @golda should I do dev track (101+) or stick with general?
**Blockers**: need to know which track
```

```
**Yesterday**: made PR for that setup bug https://github.com/org/repo/pull/42
**Today**: addressing review comments, then task 105
**Blockers**: none
```

## Bad

```
**Yesterday**: worked on stuff
**Today**: more onboarding
```
↑ what stuff? be specific

```
**Yesterday**: fixed the bug
**Today**: testing
```
↑ what bug? link to PR?

```
**Yesterday**: still stuck on same thing from Monday
**Today**: gonna keep trying
```
↑ if stuck for days ASK FOR HELP - that's a blocker

```
**Yesterday**: I have been diligently working on completing the onboarding documentation
**Today**: I plan to continue my efforts
```
↑ sounds like AI wrote it, use your actual voice

## Format

```
**Yesterday**: what you did
**Today**: what you're doing
**Blockers**: what's blocking you (or "none")
```

Be brief. Be specific. Link when helpful.
