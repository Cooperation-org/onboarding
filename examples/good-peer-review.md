# Examples of Good Peer Reviews

## Good Example: Constructive and Specific

**Work Reviewed**: Issue #5 - Leave Breadcrumbs PR

**What Worked Well**:
- Added the missing Docker setup step that wasn't in README
- Included exact command that was failing
- Linked to the docs where you found the solution
- PR was small and focused on one fix

**What Could Be Improved**:
- Could add a troubleshooting section rather than inline note
- Consider adding a "Common Issues" header for future problems
- Minor: commit message could specify which setup step (Docker vs npm)

**Skills Demonstrated**: Clear communication, leaving breadcrumbs, small focused changes

**Overall**: Strong first contribution. The troubleshooting section suggestion would make this excellent.

---

## Bad Example: Not Helpful

**Work Reviewed**: Issue #5 - Leave Breadcrumbs PR

**What Worked Well**: Looks good!

**What Could Be Improved**: Nothing, it's fine.

**Why this is bad**: Not specific, doesn't help teammate improve, doesn't show you actually reviewed the work

---

## Good Example: Critical but Fair

**Work Reviewed**: Issue #12 - Ask Questions Usefully

**What Worked Well**:
- Posted in public channel (not DM)
- Included error message
- Showed what you'd already tried

**What Could Be Improved**:
- Error message was truncated - full stack trace would help
- Missing context: which branch? which environment?
- Could include link to relevant code
- Follow-up: did you wait 30 min before pinging individuals?

**Skills Demonstrated**: Working in public, seeking help

**Overall**: Good effort on asking publicly. The additional context points would make it much easier for someone to help you quickly.

---

## Tips for Reviewers

- **Be specific**: Point to exact lines, messages, or sections
- **Show, don't just tell**: "Could add X here" vs "needs more detail"
- **Balance positive and critical**: Every review should have both
- **Focus on the work, not the person**: "This could be clearer" not "You wrote this unclearly"
- **Ask questions**: "Why did you choose X?" helps understanding
- **Link to principles**: "This follows 'Leave Breadcrumbs' well because..."
